# Pixel Dungeon Quest — デプロイ手順

レトロダンジョン探索RPG。HTML / CSS / JavaScript のみで動く静的サイトです。
ビルド不要・外部ライブラリ無し。このフォルダをそのまま静的ホスティングに上げれば公開できます。

## ファイル構成

```
pixel-dungeon-quest-web/
├── index.html   ← 入口（これが表示される）
├── style.css
├── game.js
└── assets/
    └── title.png   ← タイトル画面の背景（任意・自分で入れる）
```

`index.html` は `style.css` と `game.js` を相対パスで読み込みます。これらは必ず同じ階層に置いてください。

## タイトル画面の背景画像（任意）

タイトル画面に好きな一枚絵を使えます。

1. 使いたい画像を **`assets/title.png`** という名前で `assets` フォルダに保存する。
2. それだけでOK。読み込めればタイトル画面の背景になり、メニューが下に重なって表示されます。

- 画像が無い場合は、自動で文字ロゴ＋暗いダンジョン風の背景にフォールバックします（エラーにはなりません）。
- 推奨は横長（例：1600×900 程度）の PNG。PC では全面表示、スマホ（縦持ち）では全体が収まるバナー表示になります。
- ファイル名は必ず `title.png`。別の名前/拡張子にしたい場合は `game.js` 内の `"assets/title.png"` を書き換えてください。

## デプロイ方法（いずれか）

### GitHub Pages
1. このフォルダの中身（index.html / style.css / game.js）をリポジトリのルートに置いてプッシュ。
2. リポジトリの Settings → Pages → Branch を `main`（フォルダは `/root`）に設定して保存。
3. 数分後 `https://<ユーザー名>.github.io/<リポジトリ名>/` で公開されます。

### Netlify（最も簡単）
1. https://app.netlify.com/drop を開く。
2. この `pixel-dungeon-quest-web` フォルダ（または同梱のZIP）をブラウザにドラッグ＆ドロップ。
3. すぐに公開URLが発行されます。

### Vercel
1. このフォルダを Git リポジトリにして Vercel にインポート。
2. フレームワークは「Other（その他）」、ビルドコマンドは空のままでOK。
3. デプロイ完了で公開されます。

### ローカル確認
`index.html` をブラウザでダブルクリックするだけでも遊べます。

## 遊び方（概要）
- 移動：矢印キー / WASD（長押しで連続移動）。スマホは画面下の方向ボタン。
- マップ上の敵に触れるとバトル。各階の守護者を倒して「階段のカギ」を奪い次の階へ。
- 5階のボス「ダンジョンロード」を倒せばクリア。
- 「もちもの」で武器・防具・アクセサリーの付け替え、アイテム使用。
- セーブはブラウザの localStorage に保存（端末・ブラウザごと）。

## 注意点
- BGM・効果音はブラウザの自動再生制限により、最初のクリック/キー操作後に鳴り始めます。
- セーブデータは閲覧ブラウザの localStorage に保存されるため、別の端末・ブラウザには引き継がれません。
- HTTPS でも HTTP でも、サブディレクトリ配下でも動作します（相対パスのため）。
